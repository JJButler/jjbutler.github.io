This document describes the different APIs exposed as RESTful web services

Account API

  • Description: this endpoint allows clients to create or update accounts in Sciba
  • URL endpoint: /ws/division/$division/account/$account
  • Method: POST
  • Parameters: the json body request can specify the following parameters:
Name Type Required? Description
accountType String enumeration, valid values are: YES Account type to create or update
CLIENT_CORP: for client corporate accounts
CLIENT_PERSON: for client person accounts
COMMISSION_AGENT: for commission agent accounts
THIRD_PARTY_CORP: for third-party corporate accounts
THIRD_PARTY_PERSON: for third-party person accounts
controlAccount String representing a Sciba control account code NO Control account to link against the account
addresses List of addresses where each address is specified by: NO List of addresses associated to the account. NB: if the account already exists and a list of addresses is specified the current addresses in Sciba will be deleted and updated with the new list.
-address1: first line of address (required)
-address2: second line of address (optional)
-address3: third line of address (optional)
-city: city name (optional)
-county: county name (optional)
-country: country name (optional)
-postcode: postal code (required)
bankAccounts List of bank accounts where each bank account is specified by: NO List of bank accounts linked to the account. NB: if the account already exists and a list of bank accounts is provided in the request the current bank account list in Sciba will be deleted and updated with the new list.
-name: account’s name (required)
-sortCode: account’s sortcode (required)
-accountNumber: account’s number (required)
-description: account’s description (optional)
-bank: bank’s name (optional)
name String YES Name for the account in Sciba
email String NO Email associated to the account
title String NO Account’s holder title. NB only applies to ‘CLIENT_PERSON’ account types
firstName String NO Account’s firstname. NB only applies to ‘CLIENT_PERSON’ account types
surname String NO Account’s surname. NB only applies to ‘CLIENT_PERSON’ account types
dob String NO Account’s dob. NB only applies to ‘CLIENT_PERSON’ account types
Supported date formats are:
-yyyy-MM-dd’T’HH:mm:ss.SSSSSS’Z’ (example: 1978-01-08T00:00:00.000000Z)
-yyyy-MM-dd’T’HH:mm:ss’Z’ (example: 1978-01-08T00:00:00Z)
  • Request example:
    • Request: http://localhost:8080/ws/division/T-ST/account/TST0001
    • Method: POST
    • Body:
      {
    "accountType": "CLIENT_PERSON",
    "addresses": [
    {
        "address1": "1, Dizzy Heights",
        "address2": "The Avenue",
        "address3": "Central District",
        "city": "Somewhereville",
        "county": "Gloucestershire",
        "country": "United Kingdom",
        "postcode": "ST1 2AB"
    },
    {
        "address1": "2, Othertown",
        "postcode": "OT2 4AB"
    }
    ],
    "bankAccounts": [
    {
        "name": "PP BA 1",
        "description": "Primary bank account",
        "bank": "Santander",
        "accountNumber": "11111111",
        "sortCode": "01-02-03"
    },
    {
        "name": "PP BA 2",
        "accountNumber": "22222222",
        "sortCode": "02-03-04"
    }
    ],
    "dob": "1978-01-08T00:00:00Z",
    "email": "jb@jltgroup.com",
    "firstName": "Jeremy",
    "name": "Mr J Butler",
    "surname": "Butler",
    "title": "Mr"
}
    • Response example:
      {
    "_links": {
    "self": {
        "href": "/account/show/100001"
    }
    }
}

Policy API

  • Description: this endpoint allows clients to create or update policies in Sciba
  • URL endpoint: /ws/division/$division/account/$account/policy/$policy
  • Method: POST
  • Parameters: the json body request can specify the following parameters:
Name Type Required? Description
lpn String YES Long Policy Number which helps to track renewals for a policy
inceptionDate String YES Policy’s inception date.
Supported date formats are:
-yyyy-MM-dd’T’HH:mm:ss.SSSSSS’Z’ (example: 2014-01-08T00:00:00.000000Z)
-yyyy-MM-dd’T’HH:mm:ss’Z’ (example: 2014-01-08T00:00:00Z)
expiryDate String YES Policy’s expiry date.
Supported date formats are:
-yyyy-MM-dd’T’HH:mm:ss.SSSSSS’Z’ (example: 2015-01-08T00:00:00.000000Z)
-yyyy-MM-dd’T’HH:mm:ss’Z’ (example: 2015-01-08T00:00:00Z)
productVersion JSON object which specifies: YES Links the policy to a specific product version in Sciba
-product: product’s name (required)
-name: version’s name (required)
renewalCount Integer YES Identifies whether the policy is a NB (renewalCount == 0) or a RN (renewalCount > 0)
  • Request example:
    • Request: http://localhost:8080/ws/division/T-ST/account/TST0001/policy/POLICY0001
    • Method: POST
    • Body:
      {
    "lpn": "PTS000001",
    "inceptionDate": "2012-12-07T15:00:00.0Z",
    "expiryDate": "2013-12-07T00:00:00.0Z",
    "renewalCount": 0,
    "productVersion": {
    "product": "RET",
    "name": "1"
    }
}
    • Response example:
      {
    "_links": {
    "self": {
        "href": "/policy/show/100001"
    }
    }
}

Claim API

Insert/Update Claim

  • Description: this endpoint allows clients to create or update claims in Sciba
  • URL endpoint: /ws/division/$division/policy/$policy/claim/$claim
  • Method: PUT
  • Parameters: the json body request can specify the following parameters:
Name Type Required? Description
dateOfLoss String YES Claim’s date of loss
Supported date formats are:
-yyyy-MM-dd’T’HH:mm:ss.SSSSSS’Z’ (example: 2014-04-08T12:23:00.000000Z)
-yyyy-MM-dd’T’HH:mm:ss’Z’ (example: 2014-04-08T12:23:00Z)
claimSummary String YES Claim’s summary
claimDetails String NO Claim’s details
insured Boolean NO
totalLoss Boolean NO
state String (enumeration), valid values are: YES Claim’s state
-OPEN
-CLOSED  

  • NB: if the claim number $claim already exists in Sciba the claim record will be updated, so the current claim details will be overwritten with the new ones. Therefore, when using this endpoint it is advisable to always provide all the available details.

  • Request example:

    • Request: http://localhost:8080/ws/division/T-ST/policy/POLICY0001/claim/CLAIM0001
    • Method: PUT
    • Body:
      {
    "dateOfLoss": "2013-01-01T00:00:00.0Z",
    "claimSummary": "Camera stolen",
    "claimDetails": "Taken from back of chair while eating in restaurant.",
    "insured": true,
    "state": "OPEN"
}
    • Response example:
      {
    "_links": {
    "self": {
        "href": "/claim/show/100001"
    }
    }
}
